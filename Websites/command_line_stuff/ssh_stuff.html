SSH and Keys

On a New System with no KEYs:


make a folder called ".ssh" in your home folder

cd to ".shh"

change the permissions to 700 for folder ".ssh"

make a note of the ip address of the computer you want to ".ssh" to

type "ssh-keygen -t rsa" to make a new key

enter a passphraase. make it: "shouldidothis?"

private key is in "/home/b/.shh/id_rsa"

public key is in "/home/b/.ssh/id_rsa.pub" ... I think

send the public key to the other computer with "ssh-copy-id <username>@<host>"

in this case it will be "ssh-copy-id pi@raspberry"


If you already have the KEYs set up on your computer you can just do the: "ssh-copy-id <username>@<host>" part.

In Summary =>

mkdir ~/.shh

chmod 700 ~/.ssh

ssh-keygen -t rsa

ssh-copy-id <username>@<host>
